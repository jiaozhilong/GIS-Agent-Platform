## 3. 端到端走通验证方案

- **环境边界**：沙箱**无外网**，仅能跑编译 / 单测 / 接口级校验，**跑不了 LLM**。走通验证必须在你本地或有网环境执行（DeepSeek Key + IMA 测试库）。
- **我的交付**：保证编译通过（`mvn package` / `npm run build`）、接口级脚本、走通验证脚本（`p4_acceptance.py`）。
- **你的执行**：在真实环境 `python p4_acceptance.py`，逐功能断言；全绿即"功能全做完并走通"。
- **门槛定义（非 demo 标准）**：功能不是"页面能打开"，而是"真实数据流入 → 真实调用 → 真实产物落盘/下载"。沿用 `p3_verify.py` / `p3_acceptance.py` 的断言模式。

## 5. 风险与备注

| 风险 | 影响 | 缓解 |
